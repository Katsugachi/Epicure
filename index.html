what is Epicure?

Epicure is a family of three 300-dimensional ingredient embedding models — a compact, machine-readable map of how 1,790 food ingredients relate to each other in flavor, texture, culture, and chemistry.

Each ingredient becomes a single point in a 300-number vector space. Ingredients that co-occur in recipes, or share aroma compounds, end up geometrically close. The result: culinary intuition becomes measurable distance.

The entire matrix is roughly 2 MB — smaller than a photo. It encodes knowledge from 4.14 million multilingual recipes spanning seven languages.

Papers: Radzikowski & Chen (KAIKAKU.AI) — arXiv 2604.22776 and 2605.22391

the data files

The real Epicure data lives on Hugging Face. Each of the three model repos contains these files:

filewhat it is
embeddings.safetensorsfloat32 matrix (1790 × 300) — the actual vectors
vocab.jsonmaps ingredient name → row index
modes.jsonICA-derived culinary modes (150–200 per model)
supervised_poles.jsonunit vectors for taste / cuisine axes
config.jsonschema version, vocab size, d_model

The UMAP 2D projection (used in this viewer) lives in the companion Space:

HuggingFace Space: Kaikaku/epicure-explorer — contains umap_2d.npz (precomputed 2D coordinates) and ingredient_labels.json

Model repos: Kaikaku/epicure-cooc · Kaikaku/epicure-core · Kaikaku/epicure-chem

Interactive map: epicure-data.kaikaku.ai
MCP server: KAIKAKU-AI/epicure-mcp

three sibling models

All three share the same 1,790-ingredient vocab and Metapath2Vec training (300d, 20 epochs, skip-gram + negative sampling). They differ only in which graph edges the random walks traverse:

modelwalks usebest for
coocrecipe co-occurrence (NPMI) onlypantry pairing, cuisine clusters
coreco-occurrence + chemistry (blended)general use — start here
chemFlavorDB compound metapathsmolecular flavor bridges

The heterogeneous graph (Core/Chem) adds 2,247 typed compound nodes and 80,019 ingredient–compound edges across 15 aroma families.

15+ dimensions

The 300d space linearly encodes at least 27 continuous sensory/nutrient directions and 8 cuisine macro-regions. The paper identifies these main groups:

groupdimensions
taste (5)umami · sweet · salty · sour · bitter
heatScoville level
texture (2)crunch · moisture
geographylatitude / climate zone
processingNOVA score · fermented flag
cuisine (8)Japanese · East Asian · SE Asian · South Asian · Mediterranean · Latin American · N. Atlantic · universal

The dominant visual axis in UMAP space is sweet ↔ savory. The savoury perpendicular plane reveals clear cuisine clusters. All directions are linear probes — no fine-tuning needed.

github & index.html

To host this GUI on GitHub Pages with the real data, use this folder structure:

your-repo/
├── index.html          # this file, renamed
└── data/
    ├── umap_2d.json    # convert umap_2d.npz → JSON (see below)
    └── ingredient_labels.json

Convert the .npz to JSON once (Python):

import numpy as np, json
d = np.load("umap_2d.npz")
# d contains arrays: cooc_x, cooc_y, core_x, core_y, chem_x, chem_y
out = {k: d[k].tolist() for k in d.files}
json.dump(out, open("data/umap_2d.json", "w"))

Then update the DATA_URL constant at the top of this file to point to ./data/ instead of the HuggingFace resolve URL. Enable GitHub Pages from Settings → Pages → Deploy from branch → main / root.

CORS note: HuggingFace resolve URLs support CORS for direct browser fetch. The viewer above does this automatically — no server needed.

code examples

Load with the official Python loader:

from epicure import Epicure

m = Epicure.from_pretrained("Kaikaku/epicure-cooc")

# nearest neighbors
m.neighbors("miso", k=5)
# → [('soy_sauce', 0.91), ('fish_sauce', 0.88), ...]

# rotate toward a cuisine direction
m.slerp("miso", "cuisine:East_Asian", theta_deg=30, k=5)

# project onto a supervised axis
v = m.vec("parmesan")
pole = m.supervised_poles["taste:umami"]
score = float(v @ pole)  # cosine projection

Direct numpy (no loader):

import numpy as np, json
from safetensors.numpy import load_file

E    = load_file("embeddings.safetensors")["embeddings"]
vocab = json.load(open("vocab.json"))
itos  = {v: k for k, v in vocab.items()}
E    /= np.linalg.norm(E, axis=1, keepdims=True)  # L2 norm

def nearest(name, k=10):
    v = E[vocab[name]]
    s = E @ v
    top = np.argsort(-s)[1:k+1]
    return [(itos[i], round(float(s[i]),3)) for i in top]

print(nearest("garlic"))
